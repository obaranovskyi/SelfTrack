# Service Worker Convention

## Rule

The service worker is generated by **vite-plugin-pwa** (Workbox). It must cache the app shell and static assets so the app loads instantly and functions when offline. The update strategy is `prompt` — users are shown a notification when a new version is available and must explicitly accept the update.

---

## When to apply

Apply this convention when:

- Setting up the project for the first time
- Changing caching strategies for specific routes or assets
- Troubleshooting stale content or offline failures

---

## How to apply

### Registration — `src/main.tsx`

```typescript
import { registerSW } from 'virtual:pwa-register';

const updateSW = registerSW({
  onNeedRefresh() {
    // Show update prompt to the user — see 06_offline.tech.md
    if (confirm('New version available. Reload?')) {
      updateSW(true);
    }
  },
  onOfflineReady() {
    console.info('App is ready for offline use.');
  },
});
```

> Use a proper in-app toast/banner instead of `confirm()` — `confirm()` is used here only to illustrate the flow. See `06_offline.tech.md` for the UI pattern.

### Caching strategies

Configure Workbox runtime caching in `vite.config.ts` under `workbox.runtimeCaching`:

```typescript
VitePWA({
  registerType: 'prompt',
  workbox: {
    globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
    runtimeCaching: [
      {
        // External API responses — network-first, fall back to cache
        urlPattern: ({ url }) => url.origin !== self.location.origin,
        handler: 'NetworkFirst',
        options: {
          cacheName: 'external-api-cache',
          expiration: {
            maxEntries: 50,
            maxAgeSeconds: 60 * 60 * 24,  // 1 day
          },
          networkTimeoutSeconds: 5,
        },
      },
      {
        // Images — cache-first, long TTL
        urlPattern: /\.(?:png|jpg|jpeg|svg|gif|webp|avif)$/,
        handler: 'CacheFirst',
        options: {
          cacheName: 'images-cache',
          expiration: {
            maxEntries: 100,
            maxAgeSeconds: 60 * 60 * 24 * 30,  // 30 days
          },
        },
      },
      {
        // Google Fonts stylesheets
        urlPattern: /^https:\/\/fonts\.googleapis\.com/,
        handler: 'StaleWhileRevalidate',
        options: { cacheName: 'google-fonts-stylesheets' },
      },
      {
        // Google Fonts files
        urlPattern: /^https:\/\/fonts\.gstatic\.com/,
        handler: 'CacheFirst',
        options: {
          cacheName: 'google-fonts-webfonts',
          expiration: {
            maxEntries: 30,
            maxAgeSeconds: 60 * 60 * 24 * 365,  // 1 year
          },
        },
      },
    ],
  },
})
```

### Strategy reference

| Strategy | Best for | Behaviour |
|---|---|---|
| `CacheFirst` | Static assets, fonts, images | Serve from cache; update cache in background |
| `NetworkFirst` | API responses | Try network; fall back to cache on failure |
| `StaleWhileRevalidate` | Frequently updated but non-critical resources | Serve cache immediately; update cache in background |
| `NetworkOnly` | Non-cacheable requests (POST mutations) | Always hit network; never cache |

### Rules

- The app shell (HTML, JS, CSS, fonts) must always be precached — Workbox does this automatically via `globPatterns`.
- POST, PUT, and DELETE requests must use `NetworkOnly` — never cache mutation responses.
- Set reasonable `maxEntries` and `maxAgeSeconds` on every runtime cache to prevent unbounded cache growth.
- Never use `registerType: 'autoUpdate'` — silently swapping the SW under an active user can cause data inconsistencies. Always use `'prompt'` and let the user choose when to refresh.

## Examples

### Correct: API call wrapped in NetworkFirst

External fetch calls served from cache when offline, network when online.

### Incorrect: mutation cached

```typescript
// Never cache POST/PUT/DELETE — always NetworkOnly
urlPattern: /\/api\/users/,
handler: 'CacheFirst',  // wrong — mutations must not be cached
```
